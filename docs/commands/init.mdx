## Overview

`kanari init` creates a `config.yaml` in the current directory. It's the recommended first step after installing Kanari.

```bash
kanari init
```

```
✅ Created config.yaml
   Redis detected at redis://localhost:6379/0

   Next steps:
   kanari doctor --config config.yaml  # verify connectivity first
   kanari audit  --config config.yaml  # one-shot health check

   No Kanari account yet? Run the agent in local mode:
   kanari agent  --local --config config.yaml  # logs metrics to stdout, no API
```

---

## Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--output`, `-o` | `config.yaml` | Output file path |
| `--force`, `-f` | off | Overwrite if the file already exists |

---

## Environment variable detection

If `REDIS_URL` or `CELERY_BROKER_URL` are set in your environment, `kanari init` writes those values into the generated file instead of the localhost defaults:

```bash
export REDIS_URL=redis://prod-redis:6379/0
export CELERY_BROKER_URL=redis://prod-redis:6379/1
kanari init
# config.yaml will contain the prod URLs, not localhost
```

Supported variables:

| Variable | Used for |
|----------|----------|
| `REDIS_URL` | `redis_url` in config |
| `KANARI_REDIS_URL` | `redis_url` (alternative) |
| `CELERY_BROKER_URL` | `celery_broker_url` in config |
| `KANARI_CELERY_BROKER_URL` | `celery_broker_url` (alternative) |

---

## Redis probe

After writing the file, `kanari init` pings Redis at the configured URL:

```
✅ Created config.yaml
   Redis detected at redis://localhost:6379/0    ← reachable
```

```
✅ Created config.yaml
   ⚠️  Redis not reachable at redis://localhost:6379/0
      Update redis_url in the config file before running.
```

This is a best-effort check — it doesn't block file creation.

## Generated file

```yaml
# Kanari Agent configuration
# Generated by: kanari init

redis_url: redis://localhost:6379/0
celery_broker_url: redis://localhost:6379/0
celery_app_name: tasks

# check_interval_seconds: 30

# monitored_queues:
#   - celery
#   - priority
#   - low

thresholds:
  max_queue_size: 1000
  max_wait_time_seconds: 60
  max_task_runtime_seconds: 1800
```

Edit `redis_url` and `celery_broker_url` to match your infrastructure. All other fields are optional — Kanari's defaults cover most setups.

---

## Next steps

After `kanari init`, the recommended flow is:

```bash
kanari doctor --config config.yaml   # verify connectivity and libraries
kanari audit  --config config.yaml   # run your first health check
```

No Kanari account yet? Use local mode to run the monitoring daemon without any API calls:

```bash
kanari agent --local --config config.yaml
```
