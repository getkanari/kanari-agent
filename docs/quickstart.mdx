## Prerequisites

- Python 3.9+
- A running Redis instance
- Celery 5.2+ workers (they don't need to be healthy — Kanari will tell you if they're not)

---

## 1. Install

```bash
pip install kanari-agent
```

---

## 2. Generate a config file

```bash
kanari init
```

This creates `config.yaml` in the current directory with sensible defaults. If `REDIS_URL` or `CELERY_BROKER_URL` are already set in your environment, those values are written into the file automatically. The command also pings Redis and reports whether it's reachable:

```
✅ Created config.yaml
   Redis detected at redis://localhost:6379/0

   Next steps:
   kanari doctor --config config.yaml  # verify connectivity first
   kanari audit  --config config.yaml  # one-shot health check
```

Open `config.yaml` and update `redis_url` / `celery_broker_url` if needed, then continue.

---

## 3. Verify your setup

```bash
kanari doctor --config config.yaml
```

Checks that the required libraries are installed, Redis is reachable, and your Celery workers are responding. If anything is wrong, it tells you exactly how to fix it.

Fix what `doctor` reports before running `audit`.

---

## 4. Run a health check

```bash
kanari audit --config config.yaml
```

> **Tip:** `kanari audit` works without a config file too — it connects to `redis://localhost:6379/0` by default and prints a hint if no `--config` is provided.

## 5. Connect to Kanari for alerts (optional)

To get Slack and email alerts when something breaks, connect the agent to your Kanari account:

```bash
kanari login
```

Enter your email, click the magic link, and your API key is saved automatically.

Then configure where to send alerts:

```bash
kanari alerts configure --slack-webhook https://hooks.slack.com/...
kanari alerts configure --email you@company.com
```

Then run the continuous monitoring agent:

```bash
kanari agent
```

The agent runs in the background, collects metrics every 15 seconds, and sends you alerts the moment something goes wrong.
