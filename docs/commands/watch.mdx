---
title: kanari watch
description: Live dashboard that refreshes on a configurable interval
---

## Overview

`kanari watch` runs `kanari audit` in a loop, clearing the screen and refreshing the report on each cycle. Use it during incidents when you want a live view of what's changing.

```bash
kanari watch
```

Press `Ctrl+C` to stop.

---

## Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--config`, `-c` | — | Path to a YAML config file |
| `--interval` | `5` | Seconds between refreshes |
| `--deep`, `-d` | off | Include configuration analysis on each refresh |
| `--no-color` | off | Disable ANSI colors |

---

## Examples

```bash
# Live view, refresh every 5 seconds
kanari watch

# Faster refresh during an active incident
kanari watch --interval 2

# With config file and deep analysis
kanari watch --config config.yaml --deep --interval 10
```

---

## When to use watch vs audit

| Scenario | Command |
|----------|---------|
| Quick check before a deploy | `kanari audit` |
| CI/CD gate | `kanari audit --json` |
| Actively watching an incident unfold | `kanari watch --interval 2` |
| Checking if a fix worked | `kanari watch --interval 5` |
| Continuous background monitoring | `kanari agent` |
