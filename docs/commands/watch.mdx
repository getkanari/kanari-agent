---
title: doorman watch
description: Live dashboard that refreshes on a configurable interval
---

## Overview

`doorman watch` runs `doorman audit` in a loop, clearing the screen and refreshing the report on each cycle. Use it during incidents when you want a live view of what's changing.

```bash
doorman watch
```

Press `Ctrl+C` to stop.

---

## Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--config`, `-c` | — | Path to a YAML config file |
| `--interval` | `5` | Seconds between refreshes |
| `--deep`, `-d` | off | Include configuration analysis on each refresh |
| `--no-color` | off | Disable ANSI colors |

---

## Examples

```bash
# Live view, refresh every 5 seconds
doorman watch

# Faster refresh during an active incident
doorman watch --interval 2

# With config file and deep analysis
doorman watch --config config.yaml --deep --interval 10
```

---

## When to use watch vs audit

| Scenario | Command |
|----------|---------|
| Quick check before a deploy | `doorman audit` |
| CI/CD gate | `doorman audit --json` |
| Actively watching an incident unfold | `doorman watch --interval 2` |
| Checking if a fix worked | `doorman watch --interval 5` |
| Continuous background monitoring | `doorman agent` |
