## Overview

`kanari audit` connects to your Redis and Celery instances, collects metrics, runs the findings engine, and prints a formatted report. It then exits with a status code that reflects the severity of what it found.

```bash
kanari audit
```

---

## Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--config`, `-c` | — | Path to a YAML config file |
| `--json` | off | Output machine-readable JSON instead of TUI |
| `--md` | off | Output a Markdown report |
| `--no-config-checks` | off | Skip Redis/Celery configuration analysis (runs by default) |
| `--deep`, `-d` | — | Deprecated no-op — configuration analysis now runs by default |
| `--timeout` | `3.0` | Max seconds to wait for connections and metrics |
| `--no-color` | off | Disable ANSI colors (useful in CI logs) |

---

## Exit codes

| Code | Meaning |
|------|---------|
| `0` | Healthy — no findings |
| `1` | Warnings — at least one MEDIUM or LOW finding |
| `2` | Critical — at least one HIGH or CRITICAL finding |

## Output modes

### TUI (default)

Rich-formatted report with color-coded status indicators. Best for humans running audits interactively.

```bash
kanari audit
```

### JSON

Machine-readable output. Use this in CI scripts, log aggregation, or when piping to other tools.

```bash
kanari audit --json
```

### Markdown

Structured Markdown output. Useful for posting to Slack, GitHub issues, or incident reports.

```bash
kanari audit --md
```

## With a config file

```bash
kanari audit --config config.yaml
```

The config file lets you set custom thresholds, specify which queues to monitor, and configure privacy settings. See [Configuration](/reference/configuration) for the full reference.

---

## Configuration analysis

Every audit includes a **Configuration Analysis** section that checks your Redis and Celery settings for common production risks — no flag needed. Use `--no-config-checks` to skip it. See [Configuration Analysis](/guides/deep-audit) for details.

---

## Examples

```bash
# Quick check — is anything broken right now?
kanari audit

# CI gate — fail pipeline if critical issues found
kanari audit --json && echo "Celery healthy"

# Debug an incident, saving the full report
kanari audit --no-color 2>&1 | tee incident-$(date +%Y%m%d).log

# Check a specific environment
REDIS_URL=redis://staging-redis:6379/0 kanari audit
```
