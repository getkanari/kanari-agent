## Prerequisites

- Python 3.9+
- A running Redis instance
- Celery 5.2+ workers (they don't need to be healthy — Doorman will tell you if they're not)

---

## 1. Install

```bash
pip install doorman-agent
```

---

## 2. Point to your infrastructure

Doorman reads connection URLs from environment variables. No config file needed to get started.

```bash
export REDIS_URL=redis://your-redis:6379/0
export CELERY_BROKER_URL=redis://your-redis:6379/0
```

If your Redis is on localhost with default settings, skip this step — those are the defaults.

---

## 3. Run a health check

```bash
doorman audit
```

## 4. Enable latency tracking (recommended)

By default, Celery + Redis doesn't timestamp queued tasks, so Doorman can't measure how long tasks wait. You'll see `latency: unknown` until you add one line to your Celery app:

```python
from celery import Celery
from doorman_agent.stamps import DoormanStampPlugin

app = Celery(...)
DoormanStampPlugin.install(app)  # add this line
```

After this, Doorman shows real wait times and can alert you when tasks breach your SLA.

See [Latency Tracking](/guides/latency-tracking) for the full guide.
